Pagination

The pagination mechanism has been implemented for the investigative actions mentioned below.
  • list users
  • list user groups
  • list providers
  • list roles
The pagination mechanism has been explained below.
  • Limit Parameter: This input parameter is used to mention total number of items (valid positive integer) to be fetched in action results.
Examples
Total items in end system for reference in below examples are 950 and internally, every page fetched will have 200 result items. Based on the value of limit and other parameters combined, the required items from every page will be fetched and added to action results output. The 'after' field (for navigating to next pages) used in API calls is handled internally by the pagination mechanism.
  • Example 1: Limit = 50
    • This will fetch first 50 items

  • Example 2: Limit = 200
    • This will fetch first 200 items

  • Example 3: Limit = 240
    • This will fetch all 200 items of first page and 40 items from second page, hence in all it will fetch 240 items

  • Example 4: Limit = 1000
    • This will fetch all 950 items

  • Example 6: Limit = None
    • This will fetch all 950 items
